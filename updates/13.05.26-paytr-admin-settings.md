# PayTR Admin Settings Panel — 2026-05-13

## Objective
Allow the admin to configure PayTR API credentials (Merchant ID, Key, Salt, Test Mode) directly from the admin panel without needing to edit `.env` or restart the server.

## Files Created

### API
- `apps/api/src/utils/encryption.ts` — AES-256-GCM encrypt/decrypt helper using `SETTINGS_ENCRYPTION_KEY` from `.env`
- `apps/api/src/services/settings.ts` — settings service with 5-min in-memory cache; `getPayTRCredentials()` reads from DB and falls back to `.env` if empty
- `apps/api/src/routes/admin/settings.ts` — `GET /api/admin/settings/paytr` (returns status, never exposes raw secrets) + `PATCH` (partial update — empty fields are skipped, allows changing only testMode without re-entering key/salt)

### Frontend
- `apps/web/src/app/admin/ayarlar/page.tsx` — server component page
- `apps/web/src/app/admin/ayarlar/PayTRSettingsClient.tsx` — client form with show/hide toggles for key/salt, status badge (Yapılandırıldı / Yapılandırılmadı), live mode warning
- `apps/web/src/app/admin/ayarlar/page.module.css` — styles matching admin panel design system

## Files Modified

- `apps/api/prisma/schema.prisma` — added `SiteSetting` model (key/value, encrypted at rest in DB)
- `apps/api/src/app.ts` — mounted `adminSettingsRouter` at `/api/admin/settings` behind `requireAdmin`
- `apps/api/src/routes/payments.ts` — removed module-level `MERCHANT_ID/KEY/SALT/TEST_MODE` constants; now calls `getPayTRCredentials()` per-request (cache hit after first call) in both `/paytr/token` and `/paytr/callback` endpoints
- `apps/web/src/app/admin/layout.tsx` — added "Ayarlar" link to sidebar nav
- `.env` — auto-generated `SETTINGS_ENCRYPTION_KEY` (64-char hex, 32 bytes)
- `.env.example` — documented `SETTINGS_ENCRYPTION_KEY` with generation command

## Security Design

- Merchant Key and Salt are encrypted with AES-256-GCM before being stored in MySQL
- The master encryption key lives only in `.env` (`SETTINGS_ENCRYPTION_KEY`) — never in the DB
- `GET /api/admin/settings/paytr` returns `hasMerchantKey: boolean` and `hasMerchantSalt: boolean` — never the raw secret values
- All settings routes are behind `requireAdmin` middleware (server-side enforcement)
- `.env` fallback ensures backward compatibility — existing `.env`-based deployments keep working

## DB Change

Added `SiteSetting` model to Prisma schema. Applied to DB via `prisma db push` (non-destructive, additive only).

```prisma
model SiteSetting {
  key       String   @id @db.VarChar(191)
  value     String   @db.Text
  updatedAt DateTime @updatedAt
}
```

## How to Use

1. Go to `/admin/ayarlar`
2. Enter Merchant ID, Merchant Key, Merchant Salt
3. Choose Test Mode (Açık = test payments, Kapalı = live payments)
4. Click Kaydet
5. Key and Salt are encrypted (AES-256-GCM) and stored in DB
6. On next payment request, credentials are fetched from DB (with 5-min cache)
